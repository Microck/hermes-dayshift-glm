# kanban board design

The kanban is the human-facing interface for items that need judgment before action.

## board columns

| Column | Description |
|--------|-------------|
| **inbox** | Newly classified items, not yet reviewed by human |
| **ready** | Classifier scored and ready — has a proposed fix approach |
| **in progress** | Currently being executed by the bot |
| **blocked** | Needs more context / waiting on something |
| **done** | Fix opened as PR |
| **skip** | Human decided not to fix |

## item card

Each item on the board shows:
- Issue/PR link and title
- Source tag (nightshift / community / PR)
- Classifier scores (fixability, confidence, effort, risk)
- Proposed fix approach (one-liner generated by classifier)
- Affected files (if known)
- Attempt history (if previously tried)

## interface options

### option A: web frontend
- Simple static site served from the workspace
- API endpoint to trigger fix execution
- Same stack as daily-brief-web (which you already have)
- Most flexible, can add filtering/sorting/bulk actions

### option B: discord integration
- Structured messages with issue summary + action buttons
- Button clicks trigger the fix pipeline
- No separate UI to maintain
- Limited by Discord message formatting

### option C: markdown file
- Simplest to implement
- A `kanban.md` file in the repo, updated each cycle
- Sections for each column, items as bullet points
- Human edits the file to move items between columns
- Implementer reads the file next cycle and acts on changes
- Zero dependencies, but no real-time interaction

### recommendation

Start with option C (markdown) for the prototype. It's the fastest to ship and the interleave cycle (5h) doesn't need real-time interaction. Upgrade to option A later if needed.

## interaction flow

```
Cycle N (implement):
  1. Classifier runs → new items go to kanban inbox
  2. Bot reads kanban.md
  3. Items in "ready" column → execute fix pipeline
  4. Items in "inbox" → stay for human review
  5. Update kanban.md with new items + results

Human (anytime between cycles):
  - Reads kanban.md
  - Moves items between columns by editing the file
  - Adds notes/overrides
  - Next cycle picks up changes
```
